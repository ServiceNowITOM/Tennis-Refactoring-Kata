Tennis Score Refactoring Kata

Cleaning up grody code with lots of small refactorings

Getting the code

Code can be downloaded from Github as usual: https://github.com/ctucker/Tennis-Refactoring-Kata

  • There are implementations for multiple languages.
  • The java implementation is ready to go in Eclipse or via the pom.
  • The JS version can be run via node or via browser (see README in js directory)
  • If you're feeling adventurous, try out the Ruby or Python versions...

What we're doing

  • Refactoring!
  • Small steps, run tests after every step
  • If your IDE supports it, keep track of changes and time between tests
    • It should be no more than 60 seconds between test runs!
  • Tests should always pass. Don't move on to your next change until your last one is green.

Suggested refactorings

These might be useful refactorings to start with:

  • Rename variable/field/method
  • Extract method
  • Delete variable/field/method

Not all refactorings will be automated...you can edit the code manually, just don't change behavior or take more than 30 seconds or so to make your change!
